## 2. 시나리오 (5~10줄)
* (이곳에 한 학기 동안 개발할 프로그램의 전체 시나리오를 개괄적으로 요약합니다.)
* 
<br>
병사의 전투력은 흔히 특급전사냐, 아니냐 로 판단한다.
특급전사는 군인의 필수덕목 5개를 크게 특급, 1급, 2급으로 구분하여 5개의 종목을 모두 특급 달성 할시 특급전사라는 칭호가 수여된다. (실제로는 6급 까지 있지만 프로그램 편의를 위해 3개 등급으로 나누었다.)
종목이 많고 특급의 기준이 각기 다르기 때문에 병사들은 자기의 점수를 관리하는데 어려움을 겪는다. 나는 이 문제를 해결하고 개인 체력 관리와 목표 달성을 위해 첫째, 각자의 종목 점수를 적었을때 특급전사가 맞는지 아닌지를 판별해주는 기능과 둘째,사용한 지정한 종목을 특급달성했을때 특급전사가 될 수 있는지를 알려주는 기능 셋째, 특급전사 달성시 축하와 휴가를 지급해주는 기능이 있는 프로그램을 만드려고 한다.
## 3. 예상 기능 및 메뉴 (최소 5개)

<br>
1.병사 이름
2.팔굽혀펴기,뜀걸음, 사격, 윗몸일으키기, 정신전력 점수 입력
3.특급전사인지 아닌지 판별
4.어떤 종목을 특급이라고 가정해야 특급전사인지 판별
5.특급전사 달성이라고 판단되면 축와 휴가 지급
6.프로그램 종료

## 🟩 [2차 과제: V1.0] 입출력 + 리스트 + 조건문 - 향후 작성 예정
### **✨2차 과제 내용:**
군장병들의 가장 큰 관심사인 특급전사 선발기준을 바탕으로, 5개 과목의 점수를 입력받아, 특급전사 여부를 판별하고 포상휴가를 지급하는 실용적인 체력관리 프로그램을 기회갷ㅆ다. 단순한 조건 판별에서 나아가, 입력값의 유효성을 검사(continue, break) 하고 특급전사 달성에 실패했을 경우 어떤 과목(index와 append) 을 노력하여 특급으로 가정해야 특급전사가 될수 있는지 알려주는 목표설정 기능을 추가했다.


    
### **🤖 AI 파트너십 과정**

2. **내용 2: 효율적인 리스트 활용법 자문**

프롬프트 : 내 아이디어에서 continue와 break를 어느부분에서 적용하면 좋을지 알려줘
적용내용: 단순한 조건 판별 에서 나아가, 입력값의 유효성을 continue와 break를 사용해 검사하고 목표를 알수 있는 기능을 추가 하였다. 

프롬프트: 특급전사 달성에 실패했을때 그냥 불합격 하고 끝나는게 아니라 어떤 종목을 더 연습해야하는지 알려주고 싶어
적용내용 for i range(5): 인덱스 순회기법을 통해 특급이 아닌것만 골라낼 수 있다,
   
### **🛠️ Troubleshooting & 기술 회고:**
문제: 100점 초과시 입력을 멈췄는데도 결과가 출력
원인: 100점 넘는 값을 넣은면 break 반복문을 빠져나오게는 헀으나, 그아래에 있는 특급전사 판별코드가 멈추지 않고 그대로 실행되어 불완전한 결과를 만듦.
해결: 판별코드 전체를 if len(grades) == 5: 조건문 안에 넣어서 5개 과목을 모두 입력했을때 작동하도록 했다.

문제: 실패시 어떤 과목이 부족한지 과목 이름을 불러오지 못하는 현상
원인: 처음에는 단순하게 for grade in grades 로 등급리스트만 순회하다 보니 그 등급이 무엇을 말하는지 알수 없었음
해결: for i in range(len(subjects)): 을 사용해 인덱스 번호를 통해 뽑아내는 방식으로 바꾸었음

문제: 점수를 입력했는데 크기 비교가 안되고 에러발생
원인: input 함수로 입력받은 점수는 기본적으로 문자열로 인식되기 떄문에 조건문에서 숫자 90이나 80과 크기를 비교하려고 하니 에러가 발생함.
해결: int 함수를 씌워 입력값을 숫자로 바꾸는 형변환을 진행하였다.
